2. Descripción del Servicio

Reco es una aplicación móvil para dispositivos Android diseñada como herramienta interna de gestión para el seguimiento de rutas de reparto y cobranza de productos alimenticios. La Aplicación permite:

  • Gestión de Reparto: Registro de cantidades entregadas de productos configurables en las tiendas asignadas a cada cobrador.
  • Gestión de Cobro: Registro de cantidades vendidas, cálculo automático de montos y marcado de tiendas como cobradas o con deuda.
  • Panel de Administración: Gestión dinámica de productos, precios bases, cobradores y tiendas.
  • Exportación de Reportes: Generación de documentos PDF con el resumen detallado de cobranza por ruta.
  • Control de Acceso: Sistema de licenciamiento mensual mediante códigos de activación.
Importante: La Aplicación es una herramienta de registro y apoyo operativo. No constituye un sistema contable oficial y no sustituye la verificación física de mercancía ni los registros contables formales del negocio.

4. Funcionalidades y Alcance de la Aplicación

Con una licencia activa, el Usuario tiene acceso a las siguientes funcionalidades:

✅ El Usuario PUEDE:

  • Administrar dinámica de productos, cobradores y tiendas
  • Seleccionar su perfil de cobrador/repartidor
  • Registrar cantidades entregadas por tienda
  • Registrar cantidades vendidas por tienda
  • Marcar tiendas como repartidas o cobradas
  • Visualizar resúmenes del día
  • Generar y descargar reportes en PDF
  • Reiniciar la ruta para un nuevo día
  • Consultar el estado de su licencia
  • Extender su licencia con un nuevo código

❌ El Usuario NO PUEDE:

  • Editar tiendas ya marcadas como cobradas en el día actual
  • Registrar ventas superiores a las entregas
  • Transferir su licencia a terceros
  • Exportar o respaldar la base de datos en crudo
5. Responsabilidades del Usuario

El Usuario se compromete a:

  • Veracidad de datos: Ingresar información precisa y verídica sobre las cantidades entregadas, vendidas y cobradas.
  • Custodia del código: Mantener la confidencialidad de su código de licencia y no compartirlo con terceros.
  • Uso adecuado: Utilizar la Aplicación exclusivamente para los fines de gestión de reparto y cobro para los cuales fue diseñada.
  • Dispositivo: Mantener su dispositivo en condiciones adecuadas de funcionamiento y con sistema operativo Android compatible.

El Usuario reconoce que la Aplicación no almacena un historial de rutas anteriores. Al reiniciar la ruta, los datos previos se eliminan de forma permanente. Es responsabilidad del Usuario generar los reportes PDF antes de reiniciar.

6. Privacidad y Protección de Datos

La Aplicación maneja dos tipos de almacenamiento de datos:

a) Datos Locales (en el dispositivo):

  • Perfil del cobrador seleccionado
  • Cantidades entregadas y vendidas por tienda
  • Estado de las tiendas (repartida/cobrada) y fechas
  • Estado de membresía y fecha de vencimiento
  • Aceptación de los presentes Términos

b) Datos Remotos (servidor Supabase):

  • Código de licencia ingresado (para validación)
  • Estado de uso del código (usado/disponible)
Compromiso de privacidad: No se transmiten datos personales del Usuario ni datos operativos de reparto/cobro al servidor. La comunicación remota es exclusivamente para la validación de licencias. No se comparten datos con terceros.

La Aplicación utiliza únicamente el permiso de Internet (para validación de licencias) y acceso implícito al almacenamiento (para guardar PDFs en la carpeta de Descargas). No solicita acceso a cámara, micrófono, ubicación, contactos ni ningún otro dato sensible del dispositivo.

7. Productos y Precios Registrados

La Aplicación gestiona productos con precios unitarios bases configurables por el Usuario:

  • El Usuario, desde la sección de Administración, puede crear los productos que desee y establecerles el precio unitario correspondiente.

Estos precios y productos definidos se añadirán dinámicamente a todos los cálculos y reportes de la Aplicación. Los montos de cobro, deuda y reportes PDF se calculan automáticamente con base en los valores que el Usuario haya configurado.
